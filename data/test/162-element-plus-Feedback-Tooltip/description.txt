Component: Tooltip
Library: element-plus
Category: Feedback
Source URL: https://element-plus.org/en-US/component/tooltip

Overview:
A Vue 3 based component library for designers and developers

Props Description:

append-to
Which element the tooltip CONTENT appends to
Type:CSSSelector / HTMLElement
Default:—

effect
Tooltip theme, built-in theme: dark / light
Type:enum
Default:dark

content
Display content, can be overridden by slot#content
Type:string
Default:''

raw-content
Whether content is treated as HTML string
Type:boolean
Default:false

placement
Position of Tooltip
Type:enum
Default:bottom

fallback-placements
List of possible positions for Tooltip popper.js
Type:array
Default:—

visible / v-model:visible
Visibility of Tooltip
Type:boolean
Default:—

disabled
Whether Tooltip is disabled
Type:boolean
Default:—

offset
Offset of the Tooltip
Type:number
Default:12

transition
Animation name
Type:string
Default:—

popper-options
popper.js parameters
Type:object refer to popper.js doc
Default:{}

arrow-offset 2.9.10
Controls the offset (padding) of the tooltip’s arrow relative to the popper.
Type:number
Default:5

show-after
Delay of appearance, in millisecond, not valid in controlled mode
Type:number
Default:0

show-arrow
Whether the tooltip content has an arrow
Type:boolean
Default:true

hide-after
Delay of disappear, in millisecond, not valid in controlled mode
Type:number
Default:200

auto-close
Timeout in milliseconds to hide tooltip, not valid in controlled mode
Type:number
Default:0

popper-class
Custom class name for Tooltip's popper
Type:string
Default:—

popper-style
Custom style for Tooltip's popper
Type:string / object
Default:—

enterable
Whether the mouse can enter the tooltip
Type:boolean
Default:true

teleported
Whether tooltip content is teleported, if true it will be teleported to where append-to sets
Type:boolean
Default:true

trigger
How should the tooltip be triggered (to show), not valid in controlled mode
Type:enum / array
Default:hover

virtual-triggering
Indicates whether virtual triggering is enabled
Type:boolean
Default:—

virtual-ref
Indicates the reference element to which the tooltip is attached
Type:HTMLElement
Default:—

trigger-keys
When you click the mouse to focus on the trigger element, you can define a set of keyboard codes to control the display of tooltip through the keyboard, not valid in controlled mode
Type:Array
Default:['Enter','Space']

persistent
When tooltip inactive and persistent is false , tooltip will be destroyed
Type:boolean
Default:—

aria-label a11y
Same as aria-label
Type:string
Default:—

focus-on-target 2.11.2
When triggering tooltips through hover, whether to focus the trigger element, which improves accessibility
Type:boolean
Default:false

before-show
Triggers before tooltip is shown. Passes trigger reason as argument.
Type:Function

show
Triggers when tooltip is shown. Passes trigger reason as argument.
Type:Function

before-hide
Triggers before tooltip is hidden. Passes trigger reason as argument.
Type:Function

hide
Triggers when tooltip is hidden. Passes trigger reason as argument.
Type:Function

default
Tooltip triggering & reference element, only a single root element is accepted

popperRef
el-popper component instance
Type:object

contentRef
el-tooltip-content component instance
Type:object

isFocusInsideContent
Validate current focus event is trigger inside el-tooltip-content
Type:Function

updatePopper
Update el-popper component instance
Type:Function

Events:

onOpen
Expose onOpen function to manage el-tooltip open state
Type:Function

onClose
Expose onClose function to manage el-tooltip open state
Type:Function
