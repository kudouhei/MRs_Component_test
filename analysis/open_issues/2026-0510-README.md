# Open Issues 语料（五库 · 外部锚点）

框架 Phase 4 / RQ3 的外部锚点语料。需求定义见 [`docs/REQUIREMENTS.md`](../../docs/REQUIREMENTS.md) §5.4。

参考采集实现：`backend/scripts/collect_open_issues.py`（可替换，须满足 REQUIREMENTS 口径）。

## 有效 issue 口径

- `state=open`，且**不是** Pull Request
- **Bug 信号**：label 含 `bug` / `regression` / `type: bug` 等，或标题/正文含 bug、复现步骤等
- **排除**：enhancement、question、docs、duplicate 等标签（且无 bug 标签时）

## 组件映射

- Issue labels（`component:xxx`）、标题 `[Button]`、`fix(Button):`、正文提及评测组件名
- `matched_component` 为空表示未映射到 `data/test` 组件（仍可能是有效 bug）

## 文件

| 文件 | 说明 |
|------|------|
| `open_issues_five_repos.json` / `.csv` | 全量有效 issue |
| `open_issue_pressure_by_component.json` | 组件级 open bug 计数 |
| `open_issues_summary.json` | 汇总 |

## 运行

```bash
cd backend
python3 scripts/collect_open_issues.py
python3 scripts/collect_open_issues.py --require-component   # 仅评测组件
python3 scripts/collect_open_issues.py --max-pages 30 --library ant-design
```

建议配置根目录 `.env` 中 `GITHUB_TOKEN=...`。

## 与 RQ3

- **Exp1**：`open_issue_pressure_by_component` ↔ 当前 MR `touched_rate`
- **Exp2**：issue 标题/标签主题 ↔ 盲区 MR 类型
